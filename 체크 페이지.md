AI 도움을 받아 내 아이디어를 실제 웹사이트로 만들기

**과제 목표**
HTML/CSS/JS가 각각 어떤 역할을 하는지 설명
사용자 입력이 JS를 통해 요청(fetch)으로 바뀌고, 응답이 화면에 반영되는 흐름을 설명
Vercel Serverless Funtions가 무엇이고, 프론트에서 백엔드(Python)를 호출하는 구조를 설명
환경 변수로 API 키를 안전하게 관리해야 하는 이유 설명
로컬 환경과 배포 환경의 차이를 이해하고, 배포 후 문제를 수정 및 재배포하는 흐름을 설명
AI 코딩 도구로 코드를 생성하더라도, 오류 원인을 파악하고 수정 방향을 말로 설명

**기능 요구 사항**


1. 배포된 웹 서비스 (Vercel URL)
최소 3개 이상의 페이지 또는 섹션(메뉴 이동 가능)
반응형으로 모바일에서도 정상 표시
AI API가 연동된 기능이 1개 이상 포함(입력 -> 결과 출력)

2. Github 저장소
프로젝트 코드 업로드
프론트(HTML/CSS/JS)와 백엔드(api/)구조 분리

3. README.md
서비스 소개, 기술 스택, 실행/배포 방법, 배포 URL 포함
환경 변수(키) 설정 방법 포함

4. 서비스 기획서
서비스 목적, 타겟 사용자, 페이지 구성, 핵심 기능 정리
AI 기능의 입력/출력/실패 처리 기준 포함

5. 증빙 자료
서비스 스크린샷(데스크탑 + 모바일 + AI 기능 동작 장면) 1세트
AI 코딩 도구 사용 과정(대화 로그 또는 스크린샷) 1세
