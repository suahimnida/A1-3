AI 도움을 받아 내 아이디어를 실제 웹사이트로 만들기

**과제 목표**
HTML/CSS/JS가 각각 어떤 역할을 하는지 설명
사용자 입력이 JS를 통해 요청(fetch)으로 바뀌고, 응답이 화면에 반영되는 흐름을 설명
Vercel Serverless Funtions가 무엇이고, 프론트에서 백엔드(Python)를 호출하는 구조를 설명
환경 변수로 API 키를 안전하게 관리해야 하는 이유 설명
로컬 환경과 배포 환경의 차이를 이해하고, 배포 후 문제를 수정 및 재배포하는 흐름을 설명
AI 코딩 도구로 코드를 생성하더라도, 오류 원인을 파악하고 수정 방향을 말로 설명

**기능 요구 사항**

1) 서비스 기획
만들 웹 서비스의 아이디어 정의
목적과 타겟 사용자 정의
페이지 또는 섹션을 최소 3개 이상 설계, 메뉴 이동 방식 선정
AI 기능 1개 이상 정의(입력/출력/사용자에게 제공할 가치 포함)

2) 프로젝트 초기화 및 구조 구성
프로젝트 폴더를 생성하고 기본 구조 구성 (예: index.html, css/, js/, api/, images/, requirements.txt)
Github 저장소 생성 및 커밋 이력 리스트업

3) 프론트엔드 화면 구현(바닐라)
AI 코딩 도구를 활용해 메인 및 추가 페이지/섹션 구현
페이지/섹션 간 네비게이션(이동) 제공
기본 레이아웃과 스타일 적용

4) 반응형 적용 및 확인
모바일/태블릿/데스크톱에서 레이아웃이 깨지지 않도록 조정
최소 2가지 화면 크기에서 직접 확인

5) AI 기능 UX 최소 기준
사용자 입력을 받는 UI(폼/텍스트 입력 등) 제공
AI 결과 화면 표시
아래 실패 처리 중 최소 1개 이상을 사용자에게 안내 메시지로 제공 (빈 입력(필수값 누락), API 오류(4xx/5xx), 지연/타임아웃(응답 늦을 때))

6) AI API 연동 (백엔드: Vercel Serverless Funtions - Python)
api/ 에 Python 함수(엔드포인트) 구현
AI API(OpenAI, Claude 등) 호출 후 결과 반환
필요한 패키지를 requirements.txt에 정의
프론트에서 fetch('/api/...')로 호출하고 응답 처리

7) 배포 및 동작 검증 (Vercel)
Github와 Vercel을 연동 및 배포
배초 URL에서 전체 기능(네비게이션/반응형/AI 기능) 동작함을 확인
문제가 있으면 수정 후 재배포

8) 문서화 및 제출 패키지 구성
README.md에 핵심 내용 정리(소개/기술 스택/배포 URL/실행 방법/환경 변수)
스크린샷 및 AI 코딩 도구 사용 증빙 준비
서비스 기획서를 제츨 패키지에 포함

**개발 환경**
프론트엔드는 순수 HTML/CSS/JS로 구현(React/Vue 등 프레임워크 사용 금지)
백엔드는 Vercel Serverless Functions(Python) 사용 (api/폴더)
AI 기능 최소 1개 이상 포함

**제약 사항**
- 보안/운영 안정성
API 키는 반드시 환경 변수로 관리하고, 코드/README/스크린샷에 노출 금지
AI API는 과금/쿼터가 발생할 수 있음을 인지하고, 호출 빈도와 실패 상황 고려
키 유출이 의심되면 즉시 폐기/재발급하고, 노출된 커밋 이력도 정리

- 제출/평가 기준
#2의 "제출 패키지(필수 5종)"을 누락 없이 제출
템플릿/예시를 그대로 복제하지않기(아이디어, 문구, 구성 변경 필수)
배포 URL에서 동작이 재현 가능해야함(동료가 접속/테스트 가능)

서비스 소개, 기술 스택, 실행/배포 방법, 배포 URL 포함

서비스 목적, 타겟 사용자, 페이지 구성, 핵심 기능 정리
AI 기능의 입력/출력/실패 처리 기준 포함
